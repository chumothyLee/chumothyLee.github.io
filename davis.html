  • Abstract~*

    • Using wepagetest and Lighthouse as measurement tools I look to improve the UCD home page's performance by employing common client-side optimizations including image optimization, HTML, CSS and JS minification, and file bundling. Resulting optimizations should not perceptably change the apperance of the page relative to your reference version.

  • Results~*

    • Requests Less Data~*

      A breakdown of requests by MIME type will show that by implementing a lazy-loading scheme, the number of image bytes requested on the first view of the optimized site is drastically reduced when compared to the baseline site (1,961,031 bytes vs. 139,437 bytes)

    • Loads Faster~*

      Load time on optimized site is half that of the baseline site. On a fast network connection, the optimized site completely loads in ~5.5 seconds whereas the baseline site takes ~10.6 seconds. Slower 3g networks will show savings in the minutes.

    • Feels Faster~*

      According to reports by Google's Lighthouse tool, time to first meaningful paint, first interactive, and consistently interactive all dropped significantly (3500ms vs. 5900ms), (9200ms vs. 15000ms), and (9200ms vs. 15000ms) accordingly.
